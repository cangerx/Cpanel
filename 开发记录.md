# 开发记录

## 项目概况

当前项目是一个纯静态单页落地页，核心文件为：

- `index.html`：页面结构
- `styles.css`：页面视觉与响应式样式
- `main.js`：少量前端交互逻辑（主要是首屏视频 autoplay fallback）

本轮工作聚焦于**手机端首页首屏体验优化**，目标是在**不改 HTML 结构、不改 JS 行为、不影响桌面视觉风格**的前提下，修复窄屏设备上首屏布局失衡的问题。

---

## 本次开发背景

在 `320px–430px` 宽度区间，首页首屏原先存在以下问题：

1. 顶部导航在手机端容易拥挤。
2. Hero 文案在小屏下对固定定位过于敏感。
3. 视频区域在手机端占比过大，容易压缩文案与 CTA。
4. 小屏下容易出现视觉重心不稳，首屏信息和背景层级关系不够自然。

因此，本轮调整以 **430px 及以下** 为重点修复范围。

## 本次实现策略

### 1. 新增窄屏精修断点

在现有响应式规则基础上，增加了：

- `@media (max-width: 430px)`

该断点专门处理 320–430px 的手机端首屏问题，避免影响 `431px+` 的已有表现。

### 2. 手机端 Header 调整

针对手机端顶部区域，当前处理结果为：

- `logo` 与主导航保持**同一行**。
- 主导航位于**右侧**。
- 顶部下载按钮在手机端**隐藏**，避免破坏首行节奏。

这样可以保留顶部轻量导航，同时把主 CTA 留给 Hero 区域。

### 3. 手机端 Hero 恢复为视频背景 + 文案覆盖

在最终版本中，手机端没有继续采用“文案完全脱离视频”的流式方案，而是回到更符合当前视觉气质的处理：

- 视频仍作为首屏主背景。
- 文案层重新覆盖在视频之上。
- 文案整体位置进一步下移，压在山体上方一点的位置。

这样既保留了首屏氛围，也让标题区域和背景主体之间的关系更自然。

### 4. 手机端文案区域优化

为保证手机端标题区观感：

- 标题宽度收敛到更稳定的可读范围。
- 标题、描述、按钮采用更明确的居中对齐。
- CTA 使用统一宽度的纵向排列，点击区域稳定。

### 5. 手机端视频区域优化

最终手机端视频策略为：

- 视频作为首屏完整背景铺满 Hero 区域。
- 通过 `object-position` 和轻微 `scale` 微调，让背景主体更适合手机端裁切。
- 视频底部不再出现明显空白。

### 6. 溢出处理

在窄屏下保留：

- 横向裁切限制，避免横向滚动。
- 纵向允许自然展示，避免因为 `overflow: hidden` 掩盖布局问题。

## 关键样式方向说明

当前样式的核心取舍如下：

1. **保留桌面 overlay 风格**
   - 只在 `430px` 及以下做精修。

2. **手机端顶部减负**
   - 顶部不再保留下载按钮，减少首行挤压。

3. **手机端保持视觉氛围优先**
   - 不彻底把文案移出视频，而是保留“视频为背景、文案覆盖”的品牌气质。

4. **手机端强调首屏完整性**
   - 避免文案被裁。
   - 避免底部留白。
   - 避免横向溢出。

## 后续维护建议

后续如果继续优化手机端首屏，建议按以下顺序微调：

### 优先级 1：文案位置微调

如果需要继续贴近“文字在山上面一点点”的效果，优先调整：

- `.cp-hero-copy` 的 `top`

建议只做小幅度调整，例如 `10px–24px` 的范围，不要一次改动过大。

### 优先级 2：视频主体位置微调

如果希望山体、人物或主体元素在手机端更理想，可以继续微调：

- `.cp-hero-video` 的 `object-position`

建议优先只改纵向百分比。

### 优先级 3：标题密度微调

如果中文标题在某些机型仍显得偏满，可小幅调整：

- `.cp-hero-title` 的 `font-size`
- `.cp-hero-title` 的 `letter-spacing`
- `.cp-hero-copy` 的最大宽度

## 本次新增开发记录（首页安装与下载流程优化）

### 需求摘要

- 将首页“下载 Mac 客户端”能力拆分为：
  - 右上角：`一键下载`（直接跳转发布页）
  - 首屏主按钮：`一键安装`（弹窗展示统一安装脚本）
- 弹窗内补充海外/国内两条安装命令。
- 增加弹窗开关动效与复制命令交互。
- 修复 PC 端点击/开窗时的轻微“页面抖动”观感。

### 实现内容

1. `index.html`
- 顶部按钮改为 `一键下载`，链接到：
  - `https://github.com/cangerx/clawpanel/releases/tag/v1.0.1`
- 首屏主按钮改为 `一键安装`，触发安装弹窗。
- 新增安装弹窗结构，包含两段脚本：
  - 海外用户（GitHub）
  - 国内用户（Gitee）
- 每段命令新增 `复制命令` 按钮。

2. `main.js`
- 新增弹窗交互逻辑：
  - 打开 / 关闭
  - 点击遮罩关闭
  - `Esc` 关闭
- 新增弹窗开闭动画状态控制（进入/退出）。
- 新增复制命令逻辑（优先 `navigator.clipboard`，失败回退 `execCommand('copy')`）。
- 新增滚动条宽度补偿，避免弹窗打开时页面横向位移。

3. `styles.css`
- 新增弹窗视觉样式与移动端适配。
- 新增弹窗动效：
  - 遮罩淡入淡出
  - 对话框上浮与缩放过渡
- 新增复制按钮样式（黑银质感）与“已复制”反馈态。
- 去除主 CTA 的位移动效，改为透明度反馈，减少 PC 端“点一下会动”的感觉。

### 验证结果

- `node --check main.js` 通过。
- 交互流程符合预期：
  - 顶部 `一键下载` 直接跳转发布页（新标签页）。
  - 首屏 `一键安装` 打开弹窗。
  - 弹窗可正常关闭，复制按钮可正常反馈状态。
  - PC 端开窗时页面不再出现明显横向抖动。
